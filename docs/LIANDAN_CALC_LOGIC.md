## 二、已验证的公式（逐条对数字）

### 1. 每版印数 = ⌈数量 × 每本页数 / 每版拼数⌉ ✅ 精确命中
总页数 = 100 × 99 = 9900
- 海德堡：⌈9900/4⌉ = 2475 ✓
- 小森920：⌈9900/16⌉ = 619 ✓
- 全开：⌈9900/32⌉ = 310 ✓

> 注意：这里"每本页数"就是每本总页数（99），不是份数。三联×33份=99页。

### 2. 每版拼数 = 成品在上机尺寸上的 2D 拼版数 ✅
成品 210×140 放到 423×291（大度8开幅面）上：
- 423//210=2，291//140=2 → 2×2=4 ✓（与现有引擎 `_layout()` 逻辑一致）

### 3. 买纸数 = ⌈(每版印数 + 放数) / 每全张可开数⌉ ✅ 精确命中
海德堡：大度全开 → 大度8开 每全张开 8 张，放数=100
⌈(2475 + 100) / 8⌉ = ⌈2575/8⌉ = ⌈321.9⌉ = **322** ✓

> 关键修正：放数(开机损耗)在**印张层级 = 100**，且是先加损耗再除以每全张开数换算成全开纸。
> 现有引擎 `default_paper_loss` DB里存的是 10，**应为 100**。

### 4. 生产成本 = 纸款 + 印刷费 + 后加工费 ✅
- 印刷费 = 印版费 + 印工费（单黑1版，此例印版费并入或为0，印工费120）
- 254 + 120 + 20 = 394 ✓；284+150+20=454 ✓；327+900+20=1247 ✓

### 5. ⭐成本附加 = 生产成本 × 阶梯费率（费率随金额递减，触底10%）⭐ 重大修正
现有引擎用固定 `cost_markup_rate = 0.609`，**这是错的**。
用 2 组订单 × 3 台机器 = 6 个数据点（按生产成本排序）验证：

| 生产成本 | 成本附加 | 附加率 | 场景 |
|---|---|---|---|
| 394 | 240 | **60.9%** | 100本·海德堡(荐) |
| 454 | 246 | **54.2%** | 100本·小森920 |
| 1247 | 275 | **22.1%** | 100本·1620 |
| 2713 | 422 | **15.6%** | 1000本·小森920(荐) |
| 3090 | 309 | **10.0%** | 1000本·海德堡 |
| 3923 | 393 | **10.0%** | 1000本·1620 |

规律：**附加率随生产成本递增而单调递减，到 10% 触底**
（60.9→54.2→22.1→15.6→10.0→10.0）。

**否定了"百分比+固定值"模型**：若为 `10%+200`，则 3090、3923 应得
509、592，但实测正好是 309、393 = **纯 10%，无固定项**。
因此真实公式是纯乘性的分段费率：

```
成本附加 = 生产成本 × 阶梯费率(生产成本)
费率按金额分档，金额越大费率越低，最低档 10%（封底）
```

这对应参数页 Tab5 那张"金额区间 → 比率"阶梯表（比率列，不含固定值）。

> 现有引擎必须改：把 `生产成本 × 0.609` 换成"按生产成本金额查阶梯表 → 取该段费率 → ×费率"。
> 这是与真实逻辑差异最大、且影响每一单的一处。

#### ⚠️ 标定数据的可信性（重要方法论）

**阶梯价表的非基准行不可用于反推**。实测发现：一次报价给出的
5 行阶梯（如输入200本 → 列出 200/400/600/800/1000），只有**基准行
（你实际输入的数量）是完整真实报价**，其余 4 行是把固定成本
（印版费/开机费/设计费）按更大数量摊薄的**估算缩放值**，纸张不按整张批量重算。

证据：同为"1000本"这一数量，
- 以 1000 为基准 → 总价 3135（真实）
- 以 500 为基准的阶梯第2行(1000) → 总价 2877（缩放估算，对不上）

因此标定成本附加阶梯时，**只认「基准行 + 其成本明细弹窗」这一对数据**，
阶梯行的总价不能拿来算附加率。

**干净可信的数据点**（均为基准行 + 对应成本明细）：

| 生产成本 | 成本附加 | 附加率 | 基准数量 |
|---|---|---|---|
| 394 | 240 | 60.9% | 100本 |
| 675 | 268 | 39.7% | 200本 |
| 2713 | 422 | 15.6% | 1000本 |

趋势明确：附加率随生产成本递增单调下降，约 10% 触底。
分界点未精确标定，改造引擎时先用上述锚点近似，接入真实参数后再校准。

### 6. 选机策略 = 取总成本最低 ✅
634 < 700 < 1522 → 选海德堡。与现有引擎"多机比价取最优"一致。

### 7. 后加工：装订-胶左 ✅
100本 × 0.2元/本 = 20，最低消费(开机费)20 → max(20, 20) = **20** ✓
（对应 Tab4 后工参数：装订单价、最低消费/开机费）

## 三、对现有引擎 quote_engine.py 的修正清单

| 项 | 现状 | 应改为 | 优先级 |
|---|---|---|---|
| 成本附加 | 固定 ×0.609 | 阶梯：按生产成本查费率表→×费率（费率递减，触底10%，无固定项） | 🔴 最高 |
| 放数 default_paper_loss | 10 | 100（印张层级） | 🔴 高 |
| 买纸数换算 | 已有 ceil 逻辑 | 确认"先加放数再除每全张开数" | 🟡 核对 |
| 印版费 | 未单列 | 印刷费应拆 印版费 + 印工费 | 🟢 中 |
| 每本页数语义 | 页数 | =联数×份数（本例3×33=99），需前端联动 | 🟢 中 |

## 四、待进一步实测确认

- **纸款单价**：254/322 ≈ 0.789元/全开，与 Tab3 的 50克大度上纸 350/令(=0.7/张) 对不上，
  可能涉及"上/中/下纸"品级选择或令价换算差异，需再填不同克重/尺寸实测。
- **印工费公式**：`开机费 + 印张/1000 × 千印价` 的两个系数，需多组数据解联立。
  （1620全开印工费900而印张仅310，说明全开机开机费很高。）
- **成本附加完整阶梯**：已确认是"金额→费率"纯乘性分段、费率递减、10%触底（见第5节）。
  已知锚点：60.9%(394)、54.2%(454)、22.1%(1247)、15.6%(2713)、10%(≥3090)。
  分界点尚未精确定位，需在 500~3000 生产成本区间多扫几组订单补全。
